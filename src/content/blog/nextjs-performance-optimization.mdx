# Next.js 16 Performance Optimization: From 60 to 99 Lighthouse Score

When I first inherited a client's Next.js application, the Lighthouse score was sitting at 62. Six weeks later, we hit 99. Here's exactly what we did.

## The Starting Point

The app had these issues:
- Largest Contentful Paint (LCP): 4.8s
- Cumulative Layout Shift (CLS): 0.24
- First Input Delay (FID): 180ms
- 2.4MB JavaScript bundle

## Step 1: Implement ISR for Dynamic Pages

The biggest win came from switching product pages from client-side data fetching to Incremental Static Regeneration:

```typescript
// Before: client-side fetching (slow)
export default function ProductPage({ params }: { params: { slug: string } }) {
  const [product, setProduct] = useState(null);
  
  useEffect(() => {
    fetch(`/api/products/${params.slug}`)
      .then(r => r.json())
      .then(setProduct);
  }, []);
  
  if (!product) return <Spinner />;
  return <ProductView product={product} />;
}

// After: ISR (fast + SEO-friendly)
export async function generateStaticParams() {
  const products = await getTopProducts(); // top 1000 products
  return products.map(p => ({ slug: p.slug }));
}

export const revalidate = 3600; // revalidate every hour

export default async function ProductPage({ params }: { params: { slug: string } }) {
  const product = await getProduct(params.slug);
  return <ProductView product={product} />;
}
```

**Result**: LCP dropped from 4.8s to 1.1s for pre-generated pages.

## Step 2: Dynamic Imports for Heavy Components

```typescript
// Dynamically import 3D components and charts
const HeavyChart = dynamic(() => import('@/components/HeavyChart'), {
  ssr: false,
  loading: () => <ChartSkeleton />
});

const ThreeScene = dynamic(() => import('@/components/ThreeScene'), {
  ssr: false,
  loading: () => <div className="h-96 animate-pulse bg-white/5 rounded-xl" />
});
```

**Result**: 780KB removed from initial bundle.

## Step 3: Next/Image for Every Image

Never use a plain `<img>` tag. `next/image` gives you WebP conversion, responsive sizes, and lazy loading automatically.

```typescript
<Image
  src="/hero.jpg"
  alt="Hero"
  width={1200}
  height={600}
  priority // only for above-the-fold images
  placeholder="blur"
  blurDataURL="data:image/jpeg;base64,..."
  sizes="(max-width: 768px) 100vw, 50vw"
/>
```
